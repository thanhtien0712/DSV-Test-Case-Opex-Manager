Case Study 1 • VBA Automation

Commodity Classification Tool

Open File

The Challenge

Processing thousands of shipping records manually is prone to severe classification errors due to linguistic ambiguity in commercial documents.

Negation Traps

Declarations like "No wooden package" incorrectly triggering Wood classifications.

Substring Conflicts

The word "tea" blindly matching inside the word "Team".

Raw vs. Finished Goods

Differentiating between raw Cotton bales and finished Cotton Sweaters.

Smart Classification Engine

Phase 1: Precision
  • 1. HS Code Priority

    Automatically extracts HS Codes (e.g., 9403.60.90 -> 94036090) to map commodities with 100% accuracy before testing text.

  • 2. Dynamic Exclusions

    Reads user-defined exclusion lists (e.g., rejecting "paper bowl" from the Paper group) before any matching occurs.

Phase 2: Context
  • 3. Context-Aware Negation

    Scans 50 chars backwards and 30 forwards to detect negations like does not contain or packaging info like wooden pallet.

  • 4. Finished Goods Protection

    Identifies clothing (HS 61xx) and accessories (HS 4202) to prevent them from being misclassified as raw materials.

VBA Optimization & Best Practices

To ensure the macro processes thousands of rows instantaneously without crashing Excel, strict VBA optimization rules were applied:

1. Environment Toggling

Disables UI renders and auto-calcs to boost execution speed.

Application.ScreenUpdating = False
Application.Calculation = xlCalculationManual
Application.EnableEvents = False

2. Filter Cleansing

Prevents omitted data by clearing existing filters dynamically.

If wsShip.AutoFilterMode Then
  wsShip.AutoFilterMode = False
End If

3. In-Memory Arrays

Reads and writes bulk ranges instead of slow cell-by-cell loops.

arrShip = wsShip.Range("A2:H" & lRowShip).Value
' Process logic...
wsShip.Range("A2:H" & lRowShip).Value = arrShip
CommodityProcessor.bas 
Public Sub RunCommodityProcess()
    ' 1. Optimization
    ToggleSpeed True
    
    ' 2. Clear existing filters
    If wsCode.AutoFilterMode Then wsCode.AutoFilterMode = False
    If wsShip.AutoFilterMode Then wsShip.AutoFilterMode = False
    
    ' 3. Load Arrays
    arrShip = wsShip.Range("A2:H" & lRowShip).Value
    ReDim arrUpload(1 To lRowShip - 1, 1 To 5)
    
    ' 4. Process Logic & Validation...
    
    ' 5. Write back & format
    wsShip.Range("A2:H" & lRowShip).Value = arrShip
    
    ToggleSpeed False
End Sub

Private Sub ToggleSpeed(Optional bEnable As Boolean = True)
    With Application
        .ScreenUpdating = Not bEnable
        .Calculation = IIf(bEnable, xlCalculationManual, xlCalculationAutomatic)
        .EnableEvents = Not bEnable
    End With
End Sub

Dimension Tables (Lookup)

Keys in these tables are strictly unique. They store descriptive attributes about business entities.

Fact Tables (Event)

Keys in these tables are repeated multiple times. They record quantifiable events or transactions.
